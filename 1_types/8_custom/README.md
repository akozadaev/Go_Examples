
## Пользовательские типы (на основе существующих) - 20 мин

## 1. Зачем нужны собственные типы?

В Go можно объявить **новый тип** на основе любого существующего (базового или составного) с помощью ключевого слова `type`. Это даёт следующие преимущества:

1. **Типобезопасность (Type Safety)** - новый тип несовместим с исходным и с другими производными типами. Компилятор не позволит случайно смешать значения разных типов.
2. **Самодокументирование** - имя типа несёт смысловую нагрузку (например, `Celsius` вместо просто `float64`).
3. **Возможность добавить методы** - новый тип может иметь собственные методы, чего нельзя сделать для встроенных типов напрямую.
4. **Контроль валидации и инициализации** - можно написать конструктор (функцию `New...`), которая проверяет допустимость значения.
5. **Упрощение рефакторинга** - если изменить внутреннее представление (например, хранить температуру в миллиградусах как `int64`), интерфейс и методы останутся прежними - код, использующий тип, не сломается.

### 2. Синтаксис объявления

```go
type NewType UnderlyingType
```

Например:
```go
type Celsius float64
type Fahrenheit float64
```

Теперь `Celsius` и `Fahrenheit` - **разные типы**, хотя их базовый тип `float64` один и тот же.

### 3.3. Полный разбор примера с температурами

Ваш пример наглядно иллюстрирует все преимущества. Разберём его по частям.

#### Исходный код:
```go
package main

import "fmt"

type Celsius float64
type Fahrenheit float64

var temperature Celsius

func main() {
    temperature = 36.6
    fmt.Println(temperature)

    // ---- 1. Типобезопасность ----
    {
        var c Celsius = 25.0
        var f Fahrenheit = 77.0
        fmt.Println(c)
        fmt.Println(f)
        c = 36.6
        fmt.Println(c)
        // c = f  // ОШИБКА компиляции: cannot use f (type Fahrenheit) as type Celsius
        fahrenheit := ToFahrenheit(c)
        fahrenheit1 := c.ToFahrenheit1()
        fmt.Println(fahrenheit)
        fmt.Println(fahrenheit1)

        celsius, err := NewCelsius(-300)
        if err != nil {
            fmt.Println(err)
        }
        fmt.Println(celsius)
    }
}

// ---- 2. Явные функции для преобразования ----
func ToFahrenheit(c Celsius) Fahrenheit {
    return Fahrenheit(c*9.0/5.0 + 32.0)
}

// ---- 3. Метод для типа Celsius ----
func (c Celsius) ToFahrenheit1() Fahrenheit {
    return Fahrenheit(c*9.0/5.0 + 32.0)
}

// ---- 4. Конструктор с валидацией ----
func NewCelsius(value float64) (Celsius, error) {
    if value < -273.15 {
        return 0, fmt.Errorf("температура ниже абсолютного нуля")
    }
    return Celsius(value), nil
}
```

#### Что здесь происходит и зачем это нужно?

**1. Типобезопасность**
- `var c Celsius` и `var f Fahrenheit` - это разные типы.
- Присваивание `c = f` вызовет ошибку компиляции, даже если значения численно одинаковы. Это защищает от путаницы между шкалами (например, вы не сможете случайно передать градусы Фаренгейта в функцию, ожидающую Цельсий).

**2. Самодокументирование**
- Сигнатура `func ToFahrenheit(c Celsius) Fahrenheit` сразу говорит, что функция принимает температуру в Цельсиях и возвращает в Фаренгейтах. Читателю не нужно гадать, что означает `float64`.

**3. Добавление методов**
- Метод `(c Celsius) ToFahrenheit1()` прикреплён к типу `Celsius`. Теперь можно писать `c.ToFahrenheit1()`, что выглядит естественно (объектно-ориентированный стиль).

**4. Контроль валидации и конструктор**
- Функция `NewCelsius` проверяет, что значение не ниже абсолютного нуля. Это гарантирует, что экземпляр `Celsius` всегда будет корректным (если ошибка не возвращается). Такой паттерн используется во многих Go-пакетах.

**5. Упрощение рефакторинга**
- Если мы решим хранить температуру в микроградусах как `int64`, мы изменим только определение:
  ```go
  type Celsius int64 // храним в миллиградусах Цельсия
  ```  
  Все методы и функции, принимающие `Celsius`, останутся без изменений - код не сломается.

### 3.4. Отличие `type` от `type alias`

В Go есть также **псевдонимы типов** (type aliases), объявляемые как `type NewName = ExistingType`. Они **не создают новый тип**, а лишь дают альтернативное имя.

```go
type MyInt = int   // MyInt и int - это один и тот же тип
var a int = 5
var b MyInt = a    // OK, присваивание без приведения
```

Псевдонимы полезны для постепенного рефакторинга (например, при переносе кода между пакетами), но они **не дают типобезопасности**. В нашем примере мы использовали именно **новое определение** (`type Celsius float64`), чтобы получить все преимущества.

### 5. Когда использовать пользовательские типы?

- **Доменные сущности** - деньги, температуры, идентификаторы, валюты.
- **Единицы измерения** - метры, килограммы, секунды.
- **Состояния и статусы** - например, `type Status string` с набором констант.
- **Для улучшения читаемости и поддержки** - если вы часто путаете `int` и `string` в одном контексте.
