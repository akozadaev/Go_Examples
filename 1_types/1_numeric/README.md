## 1.1. Базовые числовые типы

| Тип | Размер (бит) | Диапазон |
|-----|-------------|----------|
| `int8`  | 8  | -128 … 127 |
| `int16` | 16 | -32768 … 32767 |
| `int32` | 32 | -2³¹ … 2³¹–1 |
| `int64` | 64 | -2⁶³ … 2⁶³–1 |
| `int`   | платформозависимый (32/64) | |
| `uint8` | 8  | 0 … 255 |
| `uint16`| 16 | 0 … 65535 |
| `uint32`| 32 | 0 … 2³²–1 |
| `uint64`| 64 | 0 … 2⁶⁴–1 |
| `uint`  | платформозависимый | |
| `float32` | 32 | ≈ ±1.18e-38 … ±3.4e38 |
| `float64` | 64 | ≈ ±2.23e-308 … ±1.8e308 |
| `complex64` / `complex128` | комплексные числа | |
| `byte`   | = `uint8` |
| `rune`   | = `int32` (Unicode code point) |

## 1.2. Zero value - значение по умолчанию
Все переменные в Go всегда инициализируются нулевым значением:
- числа → `0`
- строки → `""`
- булевы → `false`
- указатели, интерфейсы, слайсы, карты, каналы → `nil`

Это **ключевое отличие от C/C++** и гарантия безопасности.

## 1.3. Явное приведение (conversion)
В Go **нет неявного приведения** – только явное:
```go
var i int32 = 42
var j int64 = int64(i)   // OK
var k int32 = int32(j)   // OK, но потенциально опасно
```

**Опасность:** при сужении типа старшие биты отбрасываются без предупреждения.

```go
var big int64 = 3000000000   // > 2³¹
var small int32 = int32(big) // small == -1294967296 (обрезка)
```
Это молчаливая потеря данных – одна из частых ошибок.
